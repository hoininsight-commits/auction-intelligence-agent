## 0. Claude Code에게 주는 최상위 지시

당신은 이 프로젝트의 **풀스택 개발 에이전트**입니다.
아래 문서를 기준으로 **MVP를 실제로 구현 가능한 수준까지 설계하고 코드화**해야 합니다.

이번 개발의 핵심 목표는 다음과 같습니다.

1. 전국 경매·공매 물건을 통합 검색할 수 있는 백엔드 API를 만든다.
2. 온비드, 국토부 실거래가, 향후 법원경매 데이터를 수집할 수 있는 데이터 파이프라인 기반을 만든다.
3. 경매 물건, 부동산 상세, 차량 상세, 낙찰 결과, 예상 낙찰가, 관심 물건, 저장 검색 조건을 저장할 수 있는 DB 스키마를 구현한다.
4. 룰 기반 예상 낙찰가 계산 모듈을 구현한다.
5. 사용자가 검색, 상세 조회, 관심 물건 저장, 저장 검색, 기본 수익률 계산을 할 수 있는 API를 만든다.
6. 초기에는 실제 외부 API 연동이 불확실하므로, **Mock Connector + 확장 가능한 실제 Connector 구조**를 함께 구현한다.
7. 법적 리스크가 있는 크롤링은 구현하지 않는다.
8. 모든 예상 낙찰가, 수익률, 권리분석 결과는 참고용임을 명시한다.

중요 원칙:

- 먼저 동작하는 MVP를 만든다.
- 데이터 수집은 공식 API와 제휴를 우선한다.
- 대법원 경매정보 크롤링은 구현하지 말고 인터페이스만 설계한다.
- 실제 API 키가 없을 수 있으므로 `.env` 기반 설정과 Mock 데이터를 제공한다.
- 백엔드는 FastAPI로 구현한다.
- DB는 PostgreSQL 기준으로 설계하고, 로컬 개발 편의를 위해 Docker Compose를 제공한다.
- 마이그레이션은 Alembic을 사용한다.
- ORM은 SQLAlchemy 2.x 스타일을 사용한다.
- 데이터 검증은 Pydantic v2를 사용한다.
- 테스트는 pytest로 작성한다.
- 코드 구조는 추후 ML, 에이전트, 수집 작업, 관리자 기능을 확장하기 쉽게 설계한다.

# 1. 프로젝트 개요

## 1.1 프로젝트 목적

Auction Intelligence Agent는 전국의 부동산 경매, 공매, 차량 공매 정보를 통합 수집하고, 사용자가 하나의 앱에서 물건을 검색·비교·분석할 수 있도록 하는 서비스입니다.

초기 MVP는 다음 범위로 구현합니다.

```text
전국 부동산 경매·공매 검색
+ 실거래가 비교 기반
+ 룰 기반 참고 낙찰가
+ 관심 물건 저장
+ 저장 검색 조건
+ 입찰일 알림 기반 구조
+ 기본 수익률 계산기
+ 관리자 데이터 확인 API
```

차량 경매는 DB와 API 구조에는 포함하되, MVP 1차에서는 선택 기능으로 둡니다.

- 감정가

- 유사 낙찰 사례

- 기본 투자 수익률
- 권리 위험 체크리스트
- 관심 물건 알림

# 6. 데이터베이스 설계

## 6.1 핵심 원칙

DB는 다음 원칙으로 설계합니다.

1. 원본 데이터와 정규화 데이터를 분리한다.
2. 모든 외부 데이터는 `raw_source_records`에 먼저 저장한다.
3. 서비스에서 사용하는 통합 물건 정보는 `auction_items`에 저장한다.
4. 부동산과 차량 상세는 별도 테이블로 분리한다.
5. 예측 결과는 버전 관리 가능하도록 별도 테이블에 저장한다.
6. 실거래가는 별도 테이블에 저장한다.
7. 데이터 수집 작업 로그를 남긴다.
8. 추후 PostGIS를 붙일 수 있도록 위도·경도 필드를 둔다.

```json
  "total_investment": 191000000,
  "expected_sale_profit": 19000000,
  "expected_rental_income": 0,
  "interest_cost": 11250000,
  "net_profit": 7750000,
  "roi": 4.06,
  "annualized_roi": 4.06,
  "break_even_sale_price": 452250000,
  "safety_margin": 42000000,
  "disclaimer": "본 수익률 계산 결과는 사용자가 입력한 값에 기반한 단순 참고 계산이며, 세금·대출·비용·시장가격 변동에 따라 실제 결과는 달라질 수 있습니다."
}
```

계산 기준:

```text
total_cost =
취득세 + 법무비 + 명도비 + 수리비 + 중개수수료

equity =
purchase_price - loan_amount

interest_cost =
loan_amount × annual_interest_rate / 100 × holding_months / 12

total_investment =
equity + total_cost

expected_sale_profit =
expected_sale_price - purchase_price - total_cost

expected_rental_income =
monthly_rent × holding_months

net_profit =
expected_sale_price - purchase_price - total_cost - interest_cost + expected_rental_income

roi =
net_profit / total_investment × 100

annualized_roi =
roi × 12 / holding_months

break_even_sale_price =
purchase_price + total_cost + interest_cost - expected_rental_income

safety_margin =
expected_sale_price - break_even_sale_price
```

## 8.2 기본 공식


그마저도 부족한 경우:
